스탭 알바 구인 사이트 - 개인

카카오톡 오픈채팅 방에서 스탭 알바 일자리를 구하던 중 오픈채팅방의 불편한 요소들을 접한 후 이 부분들을 개선하고자 사이트를 만들게 되었다.

개요
이벤트 행사 스탭 알바 매칭 플랫폼. 매니저가 공고를 작성하고, 일반 회원/비회원이 지원하며, 관리자가 전체를 관리한다.

기술 스택

- 프론트 : nextjs 15, typescript
- styling : tailwindcss

특징

- 역할 기반 접근 제어(RBAC) : 관리자, 매니저, 일반 회원, 비회원
- 반응형 디자인
- 컴포넌트 기반 아키텍처
- mock 데이터 기반 프로토타입

## 역할 별 기능

1. 일반 회원(지원자) :

- 공고 조회 및 필터링(상태, 급여, 날짜, 지역, 카테고리)
- 관심 목록 관리
- 지원하기 (정보 선택 전송)
- 내 스케줄 관리 (카드/달력 뷰, 급여 확인, 스케줄(경력 추가 기능))
- 프로필 관리 (이미지, 경력, 자격증, 언어 등)

2. 매니저 :

- 공고 작성 (기본 양식/자유 양식, 불여넣기 파싱)
- 내 공고 관리 (공고 상태 관리)
- 지원자 관리 (상태 변경, 상세 정보 확인)
- 스케줄 관리 (카드/달력 뷰, 참여자 관리)

3. 관리자 :

- 회원 관리 (정지/해제)
- 게시글 관리 (삭제)
- 신고 관리 (처리/기각)
- 매니저 승인 관리 (승인/거절)

4. 비회원 :

- 공고 읽기 가능

## 공고 관리

- 기본 양식 : 구조화된 입력 폼
- 자유 양식 : 마크다운 에디터
- 공고문 붙여넣기 : 텍스트 파싱으로 자동 입력

## 지원 시스템

- 정보 선택 전송 : 개인 정보, 경력 등 스위치로 선택
- 지원 상태 관리 : 대기/승인/거절

## 스케줄 관리

- 카드/달력 뷰 전환
- 상태별 분류 : 예정/진행중/완료
- 참여자 정보 표시

## 프로필 관리

- 프로필 이미지 업로드 및 미리보기
- 생년월일 기반 나이 자동 계산
- 경력 추가/삭제 및 스케줄에서 불러오기
- 자격증/언어 추가/삭제
- 서류 업로드 상태 관리

## 검색 및 필터

- 공고 : 상태, 급여 범위, 날짜, 지역, 카테고리
- 관리자 대시보드 : 각 탭별 검색 및 필터
- 관심 목록 : 검색 및 최신순/오래된 순 정렬

## 트러블 슈팅

1. Hydration
   문제 상황
   서버와 클라이언트 렌더링 불일치로 Hydration 에러 발생

원인

- 로컬스토리지 접근이 서버에서 불가능
- 초기 상태가 서버/클라이언트에서 다름

해결 방법

```
const [isMounted, setIsMounted] = useState(false);

useEffect(() => {
  setIsMounted(true);
  // localStorage 접근 로직
}, []);

if (!isMounted) {
  return <div>Loading...</div>; // 또는 서버와 동일한 초기 상태
}
```

2. 반응형 디자인 구현
   문제 상황
   모바일에서 카드 레이아웃 깨짐, 텍스트 크키/여백 조정 필요
   해결 방법

- tailwindcss 커스텀 breakpoint 추가 (mobile: 390px)
- Mobile-first 접근 : 기본 모바일 스타일, sm:, md:로 확장

3. 스크롤 충돌 문제
   문제 상황
   모바일에서 목록 스크롤과 화면 스크롤이 겹침
   해결 방법
   모바일에서는 가로스크롤로 목록을 넘길 수 있도록 변경

4. 달력 카드 높이 수축 문제
   문제 상황
   모바일에서 스크롤 시 달력 카드가 수축되어 달력이 넘침
   해결 방법

- h-full 제거, min-h-[360px] sm:min-h-[420px]로 고정 최소 높이 사용
- CardContent에 고정 높이 적용

느낀점
AI 도구를 활용한 개발 경험을 통해 로직 구현보다 기획과 설계에 집중하는 것이 더 나은 제품을 만든다는 것을 배웠다.
사용자 관점에서 제품을 바라보는 능력이 향상되었고, 비즈니스 요구사항을 기술 솔루션으로 전환하는 능력을 습득하였다.
앞으로는 설계 중심 개발로 확장 가능하고 재사용 가능한 구조를 구현할 것이다.
"어떻게 만들까" 보다 "무엇을 만들까"와 "왜 만들까"에 집중할 때, 더 나은 제품이 만들어진다고 생각한다.
명확한 기획과 설계가 있으면 구현은 자연스럽게 따라올 것이라고 생각한다.
