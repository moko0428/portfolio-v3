기능

- 지도 기반 매장 탐색

  - kakao maps sdk 를 활용해 현재 위치 기준 주변 매장 표시
  - 매장 오버레이 클릭 시 매장 정보 + 해당 매장에서 판매중인 상품 리스트 다이얼로그 노출
  - Drawer 리스트와 연동되어 지도 <-> 리스트 동기화, 거리순/기본순 정렬, 키워드 필터링

- 매장 상세 & 상품 연결

  - 매장 상세 다이얼로그에서
    매장 이미지, 영업시간, 주소 등 정보 표기
  - 상품 카드를 클릭하면 상세 모달 노출

- 게시물(피드) 시스템

  - 게시물 타입
    - user_photo: 일반 사용자 사진/후기
    - admin_product: 관리자 상품 카드(가격, 색상, 사이즈, 판매 매장 연결)
  - 메인 피드에서 무한 스크롤, 좋아요, 조회수 표시
  - 공통 상세 모달
    - 게시물 본문 + 이미지 캐러셀
    - admin_product의 경우 상품 정보/판매 매장 정보 추가 표시

- 권한/역할 기반 기능 제어

  - supabase auth로 로그인/회원 관리, 프로필 테이블의 role로 권한 제어
  - manager는 본인에게 할당된 매장만 선택해 상품을 등록하도록 제한

- 조회수 좋아요 메트릭

  - supabase posts 테이블의 view_count, like_count 컬럼으로 메트릭 관리
  - 게시물 상세 모달이 열릴 때 서버 액션(incrementViewCount)을 통해 조회수 증가
  - 리스트/지도/다이얼로그 간에 조회수 값이 일관되게 보이도록 상위 상태 업데이트(onViewCountUpdate) 연동

  트러블 슈팅

  1. 카카오맵은 국내 지도만 보여준다.

  문제 상황
  외주를 맡은 매장의 특성상 외국인 손님이 국내 손님보다 월등히 높아 다른 지역(외국)에서도 맵이 열려야 하는데, 베트남에 여행을 간 지인의 말로는 맵이 열리지 않는다고 하였다.

  해결 방법
  맵을 처음 진입 시에 currentMarker를 고정값으로 설정하여 매장들이 한눈에 보일 수 있도록 하였다.
  국내에 위치할 시 현재 위치 값을 currentMarker에 설정하여 현재 위치에서 몇 km 떨어져 있는지 계산할 수 있다.

  2. 카카오맵 api 커스텀
     문제 상황
     카카오맵 api를 활용해서 지도, 마커를 사용하려 했으나 의도치 않게 쉽게 쉽게 커스텀이 힘들었다.

  해결 방법
  검색을 통해 [카카오맵 api sdk](https://react-kakao-maps-sdk.jaeseokim.dev/docs/api)를 발견하게 되었고, 지도, 마커, 오버레이 등을 비교적 쉽게 커스텀할 수 있었다.

느낀점
비개발자 클라이언트와의 소통에서 발생한 요구 불일치를 해결하기 위해 사용자의 업무 흐름을 듣고, 그 흐름에 기술을 녹여 더 효과적인 작업 방식을 제안/구현 했습니다. 이 과정에서 기술적 설명을 비개발자 관점으로 번역하는 능력과 협업 설계 능력을 강화했습니다.
